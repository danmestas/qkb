Findings

  1. Router wins on synthesis, BM25 wins on accuracy-bound vaults. Router takes 4/6 by leveraging hot.md / index.md curated reads — but those only pay off when the vault has been curated. BM25's "show me the page, don't try to be clever" wins on the AIP vault because there's no synthesis layer to ride on, and lexical-only retrieval can't hallucinate connections it didn't find.
  2. Graph won zero. Pure graph traversal is the most expensive path on average (56,970 tokens) and consistently the worst on accuracy — it invents edges (FP-1 fabricated 12 LINKS_TO entities; AIP-2 invented "Eurocontrol B2B"). The graph layer is useful as a router signal, not as a standalone retrieval strategy on these question shapes.
  3. FP-1 is a hallucination trap that survived all three skills. Despite the canonical concepts/Briefing Assembler.md being the top BM25 hit at 0.93 score, every skill answered as if Briefing Assembler were a PDF document-composition service over FAA microservices instead of an Ember Go package. Retrieval surfaced the right doc; answer-synthesis ignored it. 51k–58k tokens spent producing answers worse than no answer.
  4. Token cost is roughly flat across skills (~5% spread). The router's hot.md / index.md curated read costs about the same as the BM25 specialist's single-shot search loop, because the dominant cost is the answering agent's read + synthesis budget, not the retrieval primitive. Picking a skill on cost grounds alone is a wash.
  5. The skill cohort generalizes — three different vaults, three different winners on Q1 — and no single skill dominates everywhere. This is the v4 finding reproduced cross-vault. Validates keeping all three skills in the cohort rather than collapsing to one.
  6. Vault quality dominates skill quality. Personal vault's curated hot.md + dense wikilinks produce the bench's highest score (14/15) and lowest hallucination rate. AIP vault's thin synthesis layer pushes all three skills toward shallower answers, with bm25 winning by being honest about gaps. Invest in the wiki, not the retrieval algorithm.

Implications for the cohort

  • Keep the three-skill split. Router as default, BM25 and graph as known-shape specialists.
  • Add a hallucination-detection guardrail to the answering layer — FP-1 shows retrieval can find the right doc and the answer can still drift. Likely fix: require the answering agent to quote one verbatim sentence from each cited source before synthesizing.
  • Hold off on adding a fourth skill (e.g. PPR per RFC-0010) until the answer-synthesis hallucination floor is addressed; better retrieval doesn't help if the synthesis layer ignores it.
  • Re-run this bench against future vault-cultivation changes (hot.md hygiene, wikilink density) to see whether the per-vault skill rankings shift — vault-quality-vs-algorithm-quality is the real lever.

Methodology

18 retrieval runs dispatched as parallel subagents (one per skill × question × vault). Each subagent received only the skill's SKILL.md, the vault root, the collection name, and the question. Token usage and elapsed time were extracted from the harness <usage> block. Each vault's 6 answers were then scored by a per-vault grader agent that spot-checked claims by reading the actual vault contents via qkb and Read. Grader scores: coverage (0–5 — does it hit the key facts), accuracy (0–5 — do citations resolve, claims correct), synthesis (0–5 — does it actually answer or just list). Reproduce via bench/results/BENCH-PROMPT.md.
